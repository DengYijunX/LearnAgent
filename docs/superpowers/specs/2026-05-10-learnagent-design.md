# LearnAgent — 设计规格

## 概述

LearnAgent 是一个面向自学者（初期为单人使用）的 AI 学习助手。帮助用户完成 "发现新技术 → 理解核心概念 → 动手实践" 的完整学习闭环。聚焦 AI Agent 开发领域，后期扩展至全栈。

## 核心能力

### 1. 按需对话学习（P0 — 第一优先）

用户输入技术名词/链接/GitHub 仓库，agent 自动搜索、分析、输出结构化知识总结。

- 输入方式：技术名词、文档链接、GitHub 仓库地址
- 输出：核心概念总结、知识点列表、关联技术图谱
- 流程：Router 识别输入类型 → 搜索/抓取 → LLM 提炼 → 结构化返回

### 2. 场景化教学项目生成（P1 — 第二优先）

用户选择一个想学的技术，agent 自动生成渐进式教学场景和可运行项目。

- 生成流程（Plan-then-Execute）：
  1. 制定学习计划（场景列表、难易递增），用户确认
  2. 按步骤生成代码文件，创建本地项目目录
  3. 每一步引导用户运行、观察效果、理解原理
  4. 支持 checkpoint：暂停、回退、修改场景重试
- 体现教学深度：不只给代码，而是通过场景演进讲解设计思想和使用场景

### 3. 定时推送（P2 — 第三优先）

按用户设定时间自动推送匹配兴趣的学习内容。

- Cron 触发 → 按用户偏好搜索 → 生成摘要 → 推送到通知渠道
- 通知渠道：Slack Webhook / 飞书 Bot（初期支持一个即可）

## Agent 决策框架

### Router + 双路径模式

```
用户输入 → Router 判断复杂度
  ├── 简单任务（问概念、搜资料）→ ReAct 路径
  │     Search → Read → Summarize → Return
  │
  └── 复杂任务（生成教学项目）→ Plan-then-Execute 路径
        Plan（制定学习计划）→ 用户确认 → 分步 Execute → Checkpoint
```

- **Router**：LLM 调用判断任务类型，路由到对应路径
- **ReAct 路径**：观察 → 思考 → 行动循环，适合 1-3 步能完成的任务
- **Plan-then-Execute 路径**：参考 LangGraph 图式设计，带 checkpoint 的步骤化执行，每步可暂停/回退/重试

## 工具层

初期 5 个核心工具，函数级硬编码，不做插件系统：

| 工具 | 用途 | 依赖 |
|---|---|---|
| Web Search | 搜索技术资料 | Tavily API |
| Content Fetch | 抓取网页内容转 Markdown | Jina AI Reader |
| GitHub Discovery | 搜索仓库、查看 Trending | GitHub API |
| File System | 创建教学项目目录、写代码文件 | 本地 fs |
| Cron + Notify | 定时触发 + 推送通知 | node-cron / BullMQ + Slack/飞书 Webhook |

## 记忆层

渐进式设计，每个阶段只加必要的：

| 层级 | 存储 | 内容 | 生命周期 | 实现阶段 |
|---|---|---|---|---|
| 工作记忆 | LangGraph State | 当前会话上下文、中间步骤状态 | 单次会话 | P0 |
| 短期记忆 | SQLite + ChromaDB | 近期对话摘要、学习过的内容片段（支持 RAG 检索） | 数天~数周 | P1 |
| 长期记忆 - 用户画像 | JSON 文件 | 已掌握技能、学习偏好、难度偏好 | 永久 | P1 |
| 长期记忆 - 知识图谱 | 暂不实现 | 技术间的前置/关联关系 | — | 后期扩展 |

## 技术选型

| 层 | 选型 | 原因 |
|---|---|---|
| Agent 编排 | LangGraph (Python) | 图式设计天然适合教学流程的多步骤、有 checkpoint |
| LLM | Claude API | Sonnet 日常对话，Opus 复杂项目生成 |
| Web 框架 | FastAPI | 轻量，后续加 WebSocket 方便 |
| 向量存储 | ChromaDB | 轻量嵌入式，不需要单独服务 |
| 结构化存储 | SQLite | 零配置，够用 |
| 部署 | 本地单进程 | 初期不需要 Docker/多服务 |

## 实现优先级

1. **P0**：按需对话学习 — 最小闭环（搜 → 分析 → 总结）
2. **P1**：场景化教学项目生成 — Plan-then-Execute + Checkpoint
3. **P2**：定时推送 — Cron 触发 + 通知
4. 记忆层贯穿全程，每个阶段只加当前需要的部分

## 非目标（本期不做）

- 多用户支持（架构预留 Docker 沙箱隔离点，不实现）
- 多模态输入（图片/视频学习）
- Web UI（初期用 CLI 交互）
- 知识图谱数据库
- 插件/工具市场
- 社交媒体内容源（小红书/抖音等）
